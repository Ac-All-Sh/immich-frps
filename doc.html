1. 项目概述Overview

frps-panel 是基于 fatedier/frp 二次开发的 frp 内网穿透可视化管理面板。提供 frps 服务端与 frpc 客户端的 Web 管理界面，支持 16 种 CPU 架构，免配置文件，一键安装。适用于 Immich 照片同步等大文件传输场景的深度优化。

frps-panel is a visual management panel for frp, forked from fatedier/frp. It provides Web-based management for both frps server and frpc client, supporting 16 CPU architectures with one-click installation. Deeply optimized for large file transfer scenarios such as Immich photo sync.

3. Linux 部署Linux Deployment

3.1 通用安装步骤General Steps

从 Releases 下载对应架构的二进制文件，赋予执行权限后安装为系统服务。

Download the binary for your architecture from Releases, grant execute permission, and install as a system service.

# 下载并安装 (以 amd64 为例)Download & install (amd64 example)
wget https://github.com/attychen/immich-frps/releases/latest/download/acfrps_v0.69.1_attychen_linux_amd64
chmod +x acfrps_v0.69.1_attychen_linux_amd64
sudo ./acfrps_v0.69.1_attychen_linux_amd64 install

安装程序将交互式提示输入：绑定端口 → 管理后台端口 → 后台地址 → 用户名 → 密码。

The installer will interactively prompt for: bind port → dashboard port → dashboard address → username → password.

3.2 各架构专项说明Architecture Notes

amd64

适用绝大多数 x86_64 Linux 服务器，无需额外依赖。

For most x86_64 Linux servers. No extra dependencies.

arm64 （AArch64）

适用树莓派 4/5、Orange Pi 5、AWS Graviton 等 ARM 64 位设备。

For Raspberry Pi 4/5, Orange Pi 5, AWS Graviton, and other ARM 64-bit devices.

arm (v5) / armhf (v7)

armhf (v7) 适用树莓派 2/3 等支持硬件浮点的设备；arm (v5) 适用老旧 ARM 设备。

armhf (v7) for Raspberry Pi 2/3 and devices with hardware FP. arm (v5) for legacy ARM.

# armhf (v7)
wget https://github.com/attychen/immich-frps/releases/latest/download/acfrps_v0.69.1_attychen_linux_armhf
chmod +x acfrps_v0.69.1_attychen_linux_armhf
sudo ./acfrps_v0.69.1_attychen_linux_armhf install

mips / mipsle （路由器）

适用 MT7621、MT7620 等 MIPS 架构路由器（软浮点）。需已刷 OpenWrt/Padavan 且存储空间充足。注意：mipsle（小端）更为常见。

For MT7621, MT7620 MIPS routers (soft-float). Requires OpenWrt/Padavan with sufficient storage. Note: mipsle (little-endian) is more common.

# mipsle (小端，推荐little-endian, recommended)
wget https://github.com/attychen/immich-frps/releases/latest/download/acfrps_v0.69.1_attychen_linux_mipsle
chmod +x acfrps_v0.69.1_attychen_linux_mipsle
./acfrps_v0.69.1_attychen_linux_mipsle install

mips64 / mips64le （龙芯 3A/3B）

适用龙芯 3A/3B 处理器。请根据系统字节序选择：mips64 大端，mips64le 小端。

For Loongson 3A/3B. Choose based on endianness: mips64 = BE, mips64le = LE.

riscv64

适用 RISC-V 64 位开发板（昉·星光、SiFive HiFive 等）。

For RISC-V 64-bit boards (VisionFive, SiFive HiFive, etc.).

loong64 （龙芯 LoongArch）

适用龙芯 3A6000 等 LoongArch 架构处理器。注意：这是龙芯自主指令集，非 MIPS。

For Loongson 3A6000 and newer LoongArch processors. Note: This is Loongson's own ISA, not MIPS.

8. Docker 部署Docker Deployment

支持：linux/amd64 • linux/arm64 • linux/arm/v7

Supports: linux/amd64 • linux/arm64 • linux/arm/v7

docker run

# host 网络模式 (推荐)host network mode (recommended)
docker run -d --name frps-panel --network host --restart unless-stopped \
  -v /etc/frp:/etc/frp -v /var/log/frp:/var/log/frp \
  acallsh/frps-panel:latest

# 桥接模式bridge mode
docker run -d --name frps-panel --restart unless-stopped \
  -p 7000:7000 -p 7500:7500 -p 8080:8080 \
  -v /etc/frp:/etc/frp acallsh/frps-panel:latest

docker-compose.yml

version: '3.8'
services:
  frps-panel:
    image: acallsh/frps-panel:latest
    container_name: frps-panel
    network_mode: host
    restart: unless-stopped
    volumes:
      - /etc/frp:/etc/frp
      - /var/log/frp:/var/log/frp

9. 服务端配置Server Configuration

配置文件 /etc/frp/frps.toml，可通过 Web 面板在线修改或直接编辑。

Config file: /etc/frp/frps.toml. Editable via Web panel or directly.

bindPort = 7000
auth.token = "your_secure_token"

webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "your_password"

vhostHTTPPort = 8080
vhostHTTPSPort = 8443

10. 客户端配置Client Configuration

frpc 客户端需在 frps Web 面板中生成，二进制内嵌认证密钥，无需手动配置 token。

frpc clients are generated via the frps Web panel. The binary embeds the auth key — no manual token needed.

  1. 登录 frps Web 面板 → 用户管理 → 新增用户
  2. Login frps Web Panel → User Management → Add User
  3. 选择用户 → 生成客户端 → 下载对应架构 frpc
  4. Select User → Generate Client → Download frpc for target arch
  5. 上传至目标设备，执行 ./acfrpc_xxx install
  6. Upload to target device, run ./acfrpc_xxx install

13. 常见问题FAQ

Q: 如何确认设备架构？

Q: How to check device architecture?

uname -m
# x86_64  → amd64
# aarch64 → arm64
# armv7l  → armhf
# mips    → mips/mipsle

Q: 面板端口无法访问？

Q: Panel port not accessible?

# Linux (ufw)
sudo ufw allow 7500
# Linux (firewalld)
sudo firewall-cmd --add-port=7500/tcp --permanent && sudo firewall-cmd --reload

Q: 如何升级？

Q: How to upgrade?

Web 面板 → 在线升级（推荐），或重新下载最新二进制覆盖安装。

Web Panel → Online Upgrade (recommended), or re-download the latest binary.

Q: Docker 如何查看日志？

Q: How to view Docker logs?

docker logs -f frps-panel
